# 青鬼風 2Dアクションゲーム

シンプルな2Dアクションゲームです。青鬼（大きい四角）から逃げ続けてください。

## 遊び方

1.  リポジトリをクローンまたはダウンロードします。
2.  `index.html` をブラウザで開くか、ローカルサーバーを起動してアクセスします。
    ```bash
    # Python 3 の場合
    python3 -m http.server

    # Node.js (npx) の場合
    npx http-server
    ```
3.  ブラウザでゲーム画面が表示されます。

### 操作方法

-   **移動**: 矢印キー (`↑`, `↓`, `←`, `→`) または `WASD` キー
-   **リスタート**: ゲームオーバー後に `R` キー

## ゲームの目的

館内にいる2人の**仲間**（ピンク色の四角）を救出し、黄色い**鍵**を見つけて、緑色の**扉**から脱出してください。青鬼に捕まるとゲームオーバーです。

### エンディング

-   **トゥルーエンド**: 仲間を全員救出して脱出する。
-   **ノーマルエンド**: 仲間を一人でも救出せずに脱出する。

## 開発

-   **HTML5 Canvas**: ゲームの描画に使用
-   **JavaScript**: ゲームロジック全般
-   **CSS**:基本的なレイアウト

---
このゲームは、テキストアドベンチャーから2Dアクションゲームへ再構成されました。

## 起動方法

1. `index.html` をブラウザで開きます。
2. 画面の選択肢をクリックして進行します。
3. 3つの結末と、隠し条件を満たしたトゥルーエンドを確認できます。

## 新作ホラーゲーム『青鬼（風）』全体構成・進行フロー

本ドキュメントは、プレイヤーを最後まで引っ張るためのゲーム全体設計案です。序盤でルールを理解させ、中盤で変化と不安を与え、終盤で一気に走り切らせる構成を前提にしています。

### 進行の全体像

```text
序盤: 洋館1階・2階
	基本ルールの習得 / 仲間とのハプニング / 最初の脱出目標
		↓
中盤: 地下室・中庭・別館
	新しい脅威の投入 / 仲間の離脱 / 複雑な謎解き
		↓
終盤: 実験室・脱出口
	真相の発覚 / 連続追跡 / 脱出とエンディング
```

## 1. ゲーム進行の詳細

### 序盤: 閉ざされた洋館

**主なエリア**

- 洋館の1階・2階
- リビング、寝室、書斎などの基本的な部屋

**このパートの役割**

- プレイヤーに探索、回避、隠れるという基本ルールを覚えさせる
- 玄関が開かない状況で「外へ出られない恐怖」を印象づける
- 仲間が散り散りになり、早い段階で不安を高める

**主なギミック**

- 最初の青鬼の出現
- クローゼットや物陰への退避
- 書斎の鍵、暗号メモなどのシンプルな鍵集めと情報収集

### 中盤: 地下通路と別館

**主なエリア**

- 隠し扉からつながる地下室
- 中庭
- 洋館の裏側にある別館

**このパートの役割**

- プレイヤーが慣れてきたタイミングで、これまでの安全策を崩す
- 洋館に隠された秘密へ踏み込ませる
- 仲間との合流と離別を繰り返し、緊張の波を作る

**新しい脅威の例**

- 足が速い青鬼
- 巨体の青鬼
- クローゼットを開けてくる青鬼

**主なギミック**

- 水門の操作
- 天秤の重さ合わせ
- 複数エリアを往復する連動型の仕掛け

### 終盤: 狂気の実験室から脱出口へ

**主なエリア**

- 地下実験室
- 本館の最上階
- 脱出口へ続く最終ルート

**このパートの役割**

- 青鬼の発生原因と館の真相を明かす
- 物語の回収をしつつ、逃走の緊張感を最大化する
- 謎解きよりも判断力とルート記憶を重視する

**演出の要点**

- 最後の鍵を取った瞬間に長距離追跡を発生させる
- 隠れ場の少ない区間を増やして、逃走そのものをゲームの核にする
- プレイヤーが事前に覚えた安全ルートを正確に走れるかを試す

## 2. マルチエンディング設計

プレイ結果と回収状況によって結末が変わるようにし、再挑戦の動機を作ります。

| エンディング | 条件 | 結末の概要 |
| --- | --- | --- |
| バッドエンド | 特定の仲間を見捨てて脱出する | 逃げ切るが、強い後悔やトラウマを残す |
| ノーマルエンド | 仲間を救えず、真相も掴めないまま脱出する | 主人公のみ生還し、館は残り続ける |
| トゥルーエンド | 隠しアイテムを全回収し、仲間を全員救出して脱出する | 青鬼の呪いを解く、または根本から断つ最良の結末 |

## 3. 中だるみ防止の仕掛け

後半で作業感が出ないよう、ステージ構成と演出の両方に変化を入れます。

### 安全地帯の崩壊

- 序盤は安全だった部屋が、中盤以降は信用できなくなる
- セーブ地点や休憩地点を完全な安全ではなく、緊張の発生源に変える

### 音の演出強化

- 終盤に向かうほどBGMを強くする
- あるいは無音に近い状態を作り、足音や環境音だけで圧をかける

### 予想を裏切る猶予

- 扉を開けた直後に青鬼が立っている
- ただし一瞬だけ硬直を入れ、反射神経と判断を試す

## 4. 開発時の注意点

- 序盤は理解しやすさを優先する
- 中盤はルール破壊と変化を優先する
- 終盤は説明よりも脱出の勢いを優先する
- どの局面でも「次の一手が分かるが、安心はできない」状態を維持する

## 5. ひとことでまとめると

序盤でルールを覚えさせ、中盤で慣れを壊し、終盤で全力疾走させる構成です。プレイヤーに「まだ終わらないのか」ではなく、「あと少しで逃げ切れるかもしれない」と思わせ続けることを狙います。
